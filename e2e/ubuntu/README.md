# Ubuntu 24.04 + xfce4/xrdp on Proxmox

Fully automated provisioning of an Ubuntu 24.04 VM with xfce4 desktop and xrdp.

## Usage

```bash
./provision.sh
```

Runs from your Mac, SSHes into Proxmox (`hilbert`), creates the VM, installs the OS, then installs the desktop over SSH. Takes ~15 min total.

Connect: `open ubuntu/connect-vm996-<ip>.rdp` (generated by provision.sh)

Destroy: `ssh root@hilbert 'qm stop 996 && qm destroy 996 --purge'`

## How it works

### The autoinstall problem

Ubuntu 24.04's subiquity installer shows a "Continue with autoinstall?" confirmation prompt unless the word `autoinstall` appears on the kernel command line. You can't modify the ISO's GRUB config without `xorriso`. A previous approach used `qm sendkey` to blindly type "yes" -- fragile and unreliable.

### The solution: direct kernel boot

We bypass GRUB entirely:

1. **Extract kernel/initrd** -- mount the stock ISO, copy `casper/vmlinuz` and `casper/initrd` to `/tmp`
2. **Direct QEMU boot** -- create the VM with `--args '-kernel /tmp/ubuntu-vmlinuz -initrd /tmp/ubuntu-initrd -append "autoinstall ds=nocloud;"'`
   - `autoinstall` -- suppresses the confirmation prompt
   - `ds=nocloud;` -- tells cloud-init to find config on a local disk (our seed ISO)
3. **Seed ISO** -- a tiny ISO (labeled `cidata`) containing `user-data` with the autoinstall config, built on Proxmox via `genisoimage`
4. **Shutdown on complete** -- `shutdown: poweroff` in user-data makes the installer power off the VM instead of rebooting (a reboot would loop back into the installer since `--args` persists)
5. **Reconfigure and restart** -- script detects `stopped` status, removes `--args`, detaches both CDROMs (ide0/ide2), sets boot to disk-only, then starts the VM
6. **Desktop install over SSH** -- xfce4 + xrdp are installed via `apt-get` after the OS is up (not during autoinstall, where apt failures are fatal and hard to debug)

### What didn't work

- **`qm sendkey` loop** -- blindly typing "yes" to the confirmation prompt. Timing-dependent, broke silently.
- **Installing xfce4 during autoinstall** -- `packages:` in user-data runs apt inside the installer environment. If it fails (network issues, dependency problems), the entire install fails with no easy way to debug.
- **Waiting for guest-agent during install** -- the agent runs in the installed OS, not the installer. It never responds until the OS boots from disk.
- **Letting the installer reboot** -- with `--args` set, QEMU always uses direct kernel boot, so it re-enters the installer in a loop.
- **Deleting seed ISO before detaching** -- the EXIT trap deleted the seed ISO file while it was still attached as ide0, causing `qm start` to fail with "volume does not exist".

## Files

- `provision.sh` -- runs on Mac. Orchestrator: builds seed ISO, runs on-proxmox.sh, waits for SSH, installs desktop
- `on-proxmox.sh` -- runs on Proxmox. Creates VM with direct kernel boot, waits for install, reconfigures for disk boot
- `user-data` -- inside seed ISO. Cloud-init autoinstall config: identity, network, ssh, qemu-guest-agent
