# Gauss Hardware Baseline

`gauss` is an AZW Beelink SER8, the second NixOS host, installed on its internal
NVMe on 2026-07-23.

## Hardware

- Firmware: `SER8_P5C8V26`, dated 2024-05-23.
- CPU: AMD Ryzen 7 8845HS with Radeon 780M Graphics; 8 cores and 16 threads.
- RAM: 27 GiB.
- Internal NVMe: Crucial `CT1000P3PSSD8`, 931.5 GiB.

## NixOS Install

- Installed via the graphical (Calamares) NixOS installer: erase disk, btrfs,
  plain swap with hibernation off.
- Layout mirrors `hardy`'s shape: `/` on the top-level btrfs subvolume,
  `home`/`nix` on named subvolumes, vfat EFI `/boot`, a swap partition. The
  authoritative layout is
  [hosts/gauss/hardware-configuration.nix](../hosts/gauss/hardware-configuration.nix),
  generated by `nixos-generate-config` during the install.
- AMD-specific detection confirmed: `kvm-amd` kernel module,
  `hardware.cpu.amd.updateMicrocode`.
- Networking: DHCP reservation gives a stable LAN IP with a working
  `gauss.imetrical.com` DNS record; Tailscale is up with a manually mapped
  `gauss.ts.imetrical.com` DNS record (not Tailscale's own MagicDNS/`ts.net`
  naming).
- Bootstrap gotcha hit on the first switch from the disposable Calamares install
  to the flake config: see [bootstrap.md](bootstrap.md).

## Storage Baseline (pre-install reference)

The figures below were collected before the NixOS install, while a removable
Fedora test disk was booted on the same hardware. They remain useful as a
storage-performance reference; the Sabrent disk itself has since been unplugged
and relabeled and is no longer attached to gauss.

- `hdparm -Tt` measurements from 2026-07-07:
  - Sabrent USB disk (10 Gbps link on the SER8): `436.04 MB/sec` buffered reads.
  - Internal NVMe: `3820.29 MB/sec` buffered reads.
  - For comparison, the same Sabrent disk measured roughly `312-323 MB/sec` on
    `hardy` after its CPU throttling fix, where the USB link was only 5 Gbps.
- The SER8's baseline had healthy `amd-pstate-epp` performance mode with boost
  enabled and no Intel thermal-clamp equivalent.
